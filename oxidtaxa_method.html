Oxidtaxa — How It Works

A visual walkthrough of the IDTAXA taxonomic classification algorithm, with toy training & classification examples.

1. The Big Picture

Oxidtaxa has two phases. Training is a one-shot process that builds a model; classification runs per query and uses the model to assign a taxonomic path with per-rank confidences.

Training pipeline

Stage 1 — prepare_data

Enumerate k-mers per sequence; compute IDF weights; build taxonomy tree skeleton.

Stage 2 — build_tree

At each internal node, select the most-discriminating k-mers (features).

Stage 3 — learn_fractions

Iteratively calibrate per-node descent thresholds so self-classification hits the target rate.

→ model.bin (tree + features + fractions + IDF + inverted index)

Classification pipeline

① k-mer extraction

Unique, sorted integer k-mer IDs per query.

② bootstrap × 100

Sub-sample S = L0.47 k-mers; descend tree independently each time.

③ tally votes

Confidence at each rank = (votes for winning child) / bootstraps.

→ taxon + confidence + alternatives

3. Training, Step by Step

3aK-mer enumeration

Each sequence is converted to a sorted list of unique integer k-mer IDs. Encoding: A=0, C=1, G=2, T=3, so ACG = 0·16 + 1·4 + 2 = 6.

seq4 GGCCGGCCGG → unique sorted IDs: {21(CCG), 26(CGG), 37(GCC), 42(GGC)}

3bIDF (inverse document frequency) weights

Rare k-mers are assigned higher weight — they carry more signal.

IDF(kmer) = log( N / (1 + n_sequences_containing_kmer) )

N = 6 sequences
  IDF(ACG) = log(6 / (1+3)) = log(1.50) = 0.41   ← common, low weight
  IDF(TTA) = log(6 / (1+1)) = log(3.00) = 1.10   ← rare, high weight
  IDF(GAT) = log(6 / (1+1)) = log(3.00) = 1.10   ← species-discriminating

3cTaxonomy tree construction

An internal tree is built from the taxonomy strings. Each internal node holds the IDs of all descendant sequences and a merged k-mer profile.

At each internal node with ≥2 children, keep only the top record_kmers_fraction of k-mers by discriminative power — how asymmetrically they appear across children.

3eFraction learning (iterative calibration)

Each node learns a sampling fraction — how aggressively a bootstrap must prefer one child over its siblings in order for descent to be counted a vote. The algorithm iterates: try classifying each training sequence, observe which nodes mis-route, and adjust their fractions until the target self-classification rate (default 0.8) is reached.

┌──────────── training iteration (repeats until convergence) ───────────┐
│                                                                       │
│   for each training sequence:                                         │
│     1. sub-sample ~sqrt(L) unique kmers                               │
│     2. start at root                                                  │
│     3. at each node: score each child using its selected k-mers      │
│     4. if the winning child's score / total ≥ node.fraction:         │
│            → descend to that child                                   │
│        else:                                                         │
│            → stop (this training seq didn't reach its leaf)          │
│     5. if the descent ended at the WRONG leaf, penalise offending    │
│        nodes by nudging their fractions downward                     │
│                                                                       │
│   check: did ≥80% of training seqs reach their correct leaf?          │
│     yes → DONE                                                        │
│     no  → iterate again                                               │
└───────────────────────────────────────────────────────────────────────┘

No unclassified_* sentinel. Earlier Oxidtaxa versions (and DECIPHER's R IDTAXA) appended a synthetic trailing unclassified_Canis element to mark truncation. Today the same information is carried two other ways — the confidence list length tells you how deep classification went, and reject_reason = "below_threshold" marks the truncation. The sentinel is gone so Python and TSV surfaces match exactly; consumers that zip taxon against a fixed rank schema no longer risk treating unclassified_Canis as a species prediction.

5. Inverted Index — Why Classification Is Fast

During training Oxidtaxa builds a posting list mapping each k-mer ID to the training sequences that contain it:

K-mer ID  →  posting list
─────────────────────────
ACG(6)    →  [seq1, seq2, seq3]
CGT(27)   →  [seq1, seq2, seq3]
CGA(24)   →  [seq3]
GCC(37)   →  [seq4, seq5]
TTA(60)   →  [seq6]
...

At classify time, instead of comparing the query against every surviving reference sequence, we simply look up each query k-mer's posting list and accumulate scores.

6. Newer Parameters — Where They Hook In

Four margin/sibling/IDF knobs were added on top of the base algorithm. Three are classify-time only (no retrain) and one is a training-time change that alters the model contents.

ParamPhaseRetrain?Hooks into…
tie_margin I5 classify no Final winner-selection (step 4f — LCA clamp)
confidence_uses_descent_margin I6 classify no Every descent step (step 4c) + final confidence (step 4d)
sibling_aware_leaf I7 classify no The terminal descent step (leaf's parent), just before leaf-phase scoring
per_rank_idf I10 train yes IDF computation (step 3b) + IDF lookup during descent (step 4c)

I5tie_margin — widen the tied set

In the baseline classifier, only groups with exactly equal total scores become "winners"; any group scoring even slightly less is discarded. With tie_margin > 0, a group whose score is within (1 − tie_margin) × max joins the tied set too, feeding the LCA clamp and alternatives list.

Reference implementation (src/classify.rs:706):
  winners = {g : tot_hits[g] == max_tot}                 if tie_margin == 0
          = {g : tot_hits[g] >= max_tot · (1 − margin)}  if tie_margin  > 0

Toy effect. Same Canis query, but with scores that are near-equal instead of identical:

Grouptot_hitstie_margin = 0.00tie_margin = 0.05
C. lupus10.0winnerwinner
C. latrans9.8discarded (9.8 ≠ 10.0)winner (9.8 ≥ 10.0·0.95)
F. catus2.1discardeddiscarded (2.1 < 9.5)

Result:

tie_margin = 0.00  →  taxon=[..., Canis, Canis_lupus]  alternatives=[]
tie_margin = 0.05  →  taxon=[..., Canis]               alternatives=[Canis_latrans, Canis_lupus]
When to turn it on. When you suspect the classifier is falsely declaring a single-species winner over near-ties from congeneric references (a common false-positive mode for marker genes). Typical values: 0.02–0.10.

I6confidence_uses_descent_margin — discount near-ties along the way

At each descent node, record how decisively the winner beat the runner-up:

margin[node] = max( (top_votes − runner_up_votes) / B ,  0.1 )       # floor at 0.1
  (B = bootstraps)

final_confidence[rank_i] = vote_fraction[rank_i]  ×  Π margin[d]
                                                     d ≤ i

So a rank reached through wide-margin descents keeps its raw confidence; a rank reached through 51/49 splits gets multiplicatively discounted.

Toy effect. Suppose our query descends through these splits:

Rankwinner votes / Brunner-up / Bmargin (floor 0.1)cumulative
Root → Animalia9820.960.96
Animalia → Canidae9550.900.864
Canidae → Canis60400.200.173

Reported confidences:

                         raw conf   ×  cumulative margin   =  final conf
  Animalia      98%               ×  0.960                  =  94.1%
  Canidae       95%               ×  0.864                  =  82.1%
  Canis         92%               ×  0.173                  =  15.9%   ← big penalty
Effect. The near-tie at the genus split (60/40) pulls Canis's confidence from 92% down to 15.9%. With threshold=60, the classifier would now report only through family, not genus. The flag makes per-rank confidences sensitive to path quality, not just final vote share.

I7sibling_aware_leaf — keep close-second species in play

The problem it fixes. The baseline descent is greedy: at each node, only the single winning child is kept, everyone else is discarded. That's fine for internal splits that feed into further descent — only the winner gets to descend anyway. But at the terminal split (the parent of leaf nodes), this choice is final. A 62/55 bootstrap split between C. lupus and C. latrans confidently resolves to C. lupus even though it was almost a coin-flip. No LCA clamp, no alternatives — the runner-up is silently lost.

What the flag does. At the terminal split only, widen the candidate set to include any sibling whose bootstrap votes are ≥ 50% of B (the winner is always retained). Those candidates then all enter the leaf-phase similarity match where each candidate's reference sequences are scored against the query k-mers. The highest-scoring reference sequences across the widened candidate set feed the tie-detection logic that produces the LCA clamp and alternatives list.

Reference implementation (src/classify.rs:263):
  w_indices = [winner]                                      if off
            = {j : vote_counts[j] ≥ 0.5·B  or  j == winner} if on
            (applied only at the terminal descent — the split just
             before leaves; internal splits are unchanged)

Detailed walk-through

Setup. We descend from Root → Animalia → Canidae → Canis. The Canis node has three leaf children representing reference species:

FlagRule appliedw_indices (candidates entering leaf phase)

Step 3 — Leaf-phase similarity match. For each candidate leaf, the classifier runs parallel_match against every reference sequence behind that leaf. This produces per-reference hit scores. Call these totals tot_hits:

Query k-mers vs every reference sequence reachable from w_indices:

  sibling_aware_leaf = false:

      ├─ seq2 (C. lupus)     tot_hits = 18.1
      └─ seq3 (C. latrans)   tot_hits = 18.3   ← very close runner-up

Step 4 — Winner-group selection. References are grouped by their taxonomic group (= leaf). The classifier computes tot_hits per group, then picks winner(s):

ModeGroup totalsmax_totwinners (by tie_margin)
sibling_aware_leaf=false C. lupus: 18.4 18.4 {C. lupus}

Step 5 — Final result:

(a) sibling_aware_leaf = false, tie_margin = 0     ← default
    taxon        = [..., Canidae, Canis, C_lupus]
    confidence   = [..., 95, 92, 62]
    alternatives = []                              ← runner-up silently lost

(b) sibling_aware_leaf = true, tie_margin = 0      ← I7 alone
    taxon        = [..., Canidae, Canis, C_lupus]
    confidence   = [..., 95, 92, 62]
    alternatives = []                              ← totals aren't exactly equal
    # so even though latrans was reconsidered, it didn't *exactly* tie → no LCA

(c) sibling_aware_leaf = true, tie_margin = 0.02   ← I7 + I5 together
    taxon        = [..., Canidae, Canis]           ← LCA clamp fires
    confidence   = [..., 95, 92]
    alternatives = [Canis_latrans, Canis_lupus]    ← both reported
Key insight: I7 alone widens the candidate set, but it needs I5 (tie_margin) to actually report the near-tie. Without I5, near-tied groups still resolve to a single winner at the leaf phase. The typical configuration pairs them: sibling_aware_leaf=true with tie_margin in 0.02–0.10.

Edge cases

  • Only one child above 50%. If the winner has > 50% and nobody else does, w_indices = [winner] — same as default. No behavioral change.
  • Winner below 50%. The winner is always retained regardless of vote count (the rule is "≥ 50% OR j == winner"). Prevents a pathological case where the classifier drops every candidate.
  • Three-way near-tie (e.g., 52/51/50). All three enter leaf phase. If the top references from all three tie at max_tot, LCA clamps to the parent (Canis) and all three species land in alternatives.
  • Internal splits (e.g., Canidae → Canis vs Vulpes) are unaffected. The flag only changes the terminal split behavior because that's the only place where a single winner would be final.
  • Interaction with min_descend. The 50% threshold here is independent of min_descend (which controls whether descent continues). This check only runs once descent has already decided to enter the leaves — it's about widening the final candidate set, not about whether to descend at all.

I10per_rank_idf — IDF that adapts to the rank being split

The baseline model stores a single global IDF vector — one weight per k-mer, computed across all sequences. With per_rank_idf, training builds an IDF matrix — one row per rank depth. At classification time, the IDF row used is determined by how deep the current descent node is.

Baseline:                           With per_rank_idf:
  idf_weights[k]  (length = n_kmers)  idf_weights_by_rank[r][k]
                                      r = 0 (Kingdom)
                                      r = 1 (Phylum)
                                      ...
                                      r = 6 (Species)

The key move in compute_idf_by_rank: at rank depth r, group training sequences by their taxonomic prefix up to that depth, then compute:

IDF_r(k)  =  log( N_r / (1 + n_groups_at_depth_r_containing_k) )

Toy example. Take k-mer ACG in our 6-sequence dataset. It appears in seq1, seq2, seq3 — all Animalia; all Canidae; all Canis; split between C. lupus (seq1,2) and C. latrans (seq3).

Rank depthN_r = distinct prefixesPrefixes containing ACGIDF_r(ACG)
0 — Kingdom2 (Animalia, Plantae)1 (Animalia)log(2/2) = 0.00 weak
1 — Family3 (Canidae, Felidae, Rosaceae)1 (Canidae)log(3/2) = 0.41
2 — Genus3 (Canis, Felis, Rosa)1 (Canis)log(3/2) = 0.41
3 — Species4 species2 (C. lupus, C. latrans)log(4/3) = 0.29 weak
Reading this:
  • At the Kingdom split, ACG is useless (IDF=0) — it only appears in one kingdom anyway.
  • At Family/Genus, ACG is moderately informative — it discriminates Canidae/Canis from the others.
  • At Species, ACG is noise again — both Canis species have it.
The global-IDF baseline would use a single value for ACG everywhere, misweighting it at multiple ranks at once. 
At classify time (src/classify.rs:508):
  let rank_idx = descent_depth;                         // which row?
  counts = &idf_weights_by_rank[rank_idx]               // use that row's weights

  # older models without the matrix fall back to the global vector automatically
  # via #[serde(default)]  — no breaking change for pre-v0.3.0 bincode files

7. The Core Insight

Bootstrapping on sub-sampled k-mers converts raw k-mer scoring into a voting system, and the vote share becomes a natural per-rank confidence measurement. Everything else — feature selection, IDF weighting, fraction learning, the inverted index — exists to make that voting system fast, calibrated, and discriminative.

Oxidtaxa — How It Works

A visual walkthrough of the IDTAXA taxonomic classification algorithm, with toy training & classification examples. Every parameter is shown at the point where it hooks into the flow.

1. The Big Picture

Oxidtaxa has two phases. Training is a one-shot process that builds a model; classification runs per query and uses the model to assign a taxonomic path with per-rank confidences.

Training pipeline

Stage 1 — prepare_data

Enumerate k-mers per sequence; compute per-rank IDF matrix; build taxonomy tree skeleton.

Stage 2 — build_tree

At each internal node, select the most-discriminating k-mers (features).

Stage 3 — learn_fractions

Iteratively calibrate per-node descent thresholds so self-classification hits the target rate.

→ model.bin (tree + features + fractions + per-rank IDF + inverted index)

Classification pipeline

① k-mer extraction

Unique, sorted integer k-mer IDs per query.

② bootstrap × 100

Sub-sample S = L0.47 k-mers; descend tree independently each time; record margins.

③ tally & report

Confidence per rank; LCA-clamp on tied groups; emit alternatives.

→ taxon + confidence + alternatives

3bIDF weights — per rank, not global

Rare k-mers should carry more signal than common ones. Oxidtaxa computes IDF per rank depth: a k-mer can be informative at genus but noise at kingdom.

For each rank depth r (Kingdom=0, Phylum=1, ..., Species=last):
  group training sequences by their taxonomic prefix up to depth r
  assign each sequence weight w_i = 1 / (sequences in its prefix group)
  c = Σ w_i  over sequences that contain the k-mer
  IDF_r(kmer) = ln( N_r / (1 + c) )                  # src/training.rs:685

The weighting means each prefix group contributes at most 1 to c — and less when only a fraction of its sequences have the k-mer. That's important when a clade is unevenly represented.

Worked example — k-mer ACG. It appears in seq1, seq2, seq3. At kingdom level, that's 3 out of 5 Animalia sequences — so Animalia contributes 3/5, not 1.

RankGroup(s) containing ACGc (weighted sum)N_rIDF_r(ACG) = ln(N_r / (1+c))
KingdomAnimalia: 3 of 5 seqs3·(1/5) = 0.62ln(2/1.6) ≈ 0.22
FamilyCanidae: 3 of 3 seqs3·(1/3) = 1.03ln(3/2) ≈ 0.41
GenusCanis: 3 of 3 seqs3·(1/3) = 1.03ln(3/2) ≈ 0.41
SpeciesC. lupus: 2/2, C. latrans: 1/12·(1/2) + 1·(1/1) = 2.04ln(4/3) ≈ 0.29 weak
Why Kingdom isn't zero: it would be if every Animalia sequence had ACG. But only 3 of 5 do, so Animalia "partially" carries ACG and the k-mer still has some kingdom-level discriminative power. The simpler formula ln(N_r / (1 + n_prefixes_containing)) only gives the right answer when every sequence inside a prefix group shares the k-mer.

I10 per_rank_idf — always on in v0.3.0+

The model stores an IDF matrix (one row per rank) instead of a single global vector. At classification time the row used is determined by the descent node's depth.

idf_weights_by_rank[0]  →  used at Kingdom split  (e.g. ACG weight = 0.00)
idf_weights_by_rank[1]  →  used at Family  split  (e.g. ACG weight = 0.41)
idf_weights_by_rank[2]  →  used at Genus   split  (e.g. ACG weight = 0.41)
idf_weights_by_rank[3]  →  used at Species split  (e.g. ACG weight = 0.29)

Source: src/training.rs:313 (compute), src/classify.rs:504 (lookup).

3eFraction learning (iterative calibration)

Each node learns a sampling fraction — how aggressively a bootstrap must prefer one child over its siblings for descent to count. Iterate until the self-classification rate hits training_threshold (default 0.8).

for each training sequence:
  1. subsample ~sqrt(L) unique kmers
  2. start at root
  3. at each node: score each child, descend if winner/total ≥ node.fraction
  4. if descent ended at WRONG leaf, nudge offending node fractions down
check: did ≥80% of seqs reach their correct leaf?
  yes → DONE    no → iterate

4cTree descent & per-step margins

For each bootstrap, we walk from Root toward leaves. At each node, every child accumulates votes across the bootstrap's k-mer draws. Whether to descend depends on the winning child's share of votes.

Single-bootstrap walk (Bootstrap #1, sample = {ACG, CGT})

  Root                       score(Animalia) = IDF₀(ACG) + IDF₀(CGT) = 0.00 + 0.69 = 0.69
  ├── Animalia               score(Plantae)  = 0
  └── Plantae                                       ← descent requires
  winner = Animalia                                    winner/total ≥ min_descend (0.98)
  0.69 / 0.69 = 1.00  ≥ 0.98  →  DESCEND
                             ▼
  Animalia → Canidae → Canis   (all straightforward)
                             ▼
  Canis  (feature kmers {CGT, GTA, CGA, GAT, ATC};  using IDF₂ row)
  ├── C. lupus    matched: {CGT, GTA}      → 2 hits  → wins this bootstrap
  └── C. latrans  matched: {CGT}           → 1 hit

Across 100 bootstraps, each node tallies vote_counts per child. A typical aggregate:

at the Canis node, after 100 bootstraps:
  vote_counts[C. lupus]    = 62
  vote_counts[C. latrans]  = 55       ← close runner-up
  vote_counts[C. familiaris] = 12
  # (sums > 100 because bootstraps can tie and vote for multiple)

I6 confidence_uses_descent_margin

At each descent step, record how decisively the winner beat the runner-up:

margin[node_i] = max( (top_votes − runner_up_votes) / B ,  0.1 )

Scoping rule: the margin recorded at node i is the decisiveness of the single decision that selects rank i+1. Its discount applies only to rank i+1 — no compounding across ranks. Each rank's confidence is penalized by the one margin that chose it, and by nothing else.

for each rank i > 0 (Kingdom, Phylum, ...):
    confidence[i]  *=  margin[i − 1]       # the margin of the decision
                                           # that selected THIS rank, only

Why non-cumulative. Margin is a decisiveness score, not a probability — multiplying them across ranks has no clean statistical meaning, it just compounds penalties. The raw vote-fraction confidence already encodes path dependence (a bootstrap only reaches rank i if it passed every prior split correctly). Compounding margins on top double-counts path uncertainty and collapses deep ranks to near-zero (e.g. 0.3⁶ ≈ 0.0007 for six 60/40 splits).

Descent steptop/Brunner/Bmargin recordedapplied to…
at Root, picks Kingdom9820.96Kingdom only
at Animalia, picks Family9550.90Family only
at Canidae, picks Genus60400.20Genus only

So with raw confidences [100, 98, 95, 92]:

Root:    100  × 1      = 100.0    # no selecting decision
Kingdom:  98  × 0.96   =  94.1    # × Root→Kingdom margin only
Family:   95  × 0.90   =  85.5    # × Animalia→Family margin only
Genus:    92  × 0.20   =  18.4    # × Canidae→Genus margin only
                                    (Genus penalty does NOT propagate down;
                                     Species below would be multiplied by
                                     its own margin, not Genus's.)

Source: src/classify.rs:230 (greedy record), src/classify.rs:388 (beam record), src/classify.rs:744 (apply). The beam path now carries a per-candidate descent_margins Vec so I6 is honored for beam_width > 1.

4dTerminal descent step & sibling widening

At the last descent step (the split whose children are leaves), the baseline picks a single winner. That decision is final — no further descent, no chance for a close runner-up to be reconsidered.

baseline behavior:
  w_indices = [winner]                           # everyone else discarded
              ↓
  only this leaf's reference sequences enter the leaf-phase match

I7 sibling_aware_leaf

At the terminal step only, widen the candidate set:

w_indices = {j : vote_counts[j] ≥ 0.5·B  or  j == winner}

Applied to our running example (B=100, votes: C. lupus=62, C. latrans=55, C. familiaris=12):

4eLeaf-phase scoring & winner groups

For every candidate in w_indices, the classifier runs a k-mer match against each individual reference sequence behind that leaf. Scores are grouped by taxonomic leaf; the best group wins.

Per-reference hits (our example, with I7 on):
  seq1 (C. lupus)     tot_hits = 18.4   ← global max
  seq2 (C. lupus)     tot_hits = 18.1
  seq3 (C. latrans)   tot_hits = 18.3   ← very close runner-up

Group totals:
  C. lupus group      max over members = 18.4
  C. latrans group    max over members = 18.3

I5 tie_margin

Baseline keeps only groups with exactly max score. With tie_margin > 0, near-tied groups also qualify:

winners = {g : tot_hits[g] == max_tot}                   if tie_margin == 0
        = {g : tot_hits[g] ≥ max_tot · (1 − tie_margin)} if tie_margin  > 0
Settingcutoffwinners
tie_margin = 018.40 exactly{C. lupus}
tie_margin = 0.02≥ 18.03{C. lupus, C. latrans}

When |winners| > 1, the classifier caps the reported taxon at the winners' LCA and lists every winner in alternatives. Source: src/classify.rs:706.

I7 and I5 together: I7 brings C. latrans into the match; I5 lets the close score actually count. One without the other usually changes nothing — the typical configuration enables both.

How the knobs interact: I7 widens candidates, I5 widens winners, I6 discounts confidence along near-tied descents. Individually each is mild — together they let the classifier honestly report "we got close, but we can't commit below genus, and here are the candidates you should consider."

7. The Core Insight

Bootstrapping on sub-sampled k-mers converts raw k-mer scoring into a voting system, and the vote share becomes a natural per-rank confidence measurement. The margin/sibling/tie knobs (I5, I6, I7) sharpen the system's ability to be honest about uncertainty — they don't change what the classifier knows, only how generously it reports when the evidence is close. The per-rank IDF (I10) complements them by making sure k-mer weights are appropriate for which rank is being split, rather than being the same at every level.

Oxidtaxa — a Rust rewrite of IDTAXA (Murali, Bhargava & Wright 2018) with per-query independent PRNG, per-rank IDF, margin/sibling-aware classification, and inverted-index acceleration.

Optimizations applied (16 total):
  Phase 2 - Algorithmic:
  1. HashSet for u_classes/all_taxa dedup (training O(n^2) -> O(n))
  2. HashMap for sequences_per_node (training O(n*t) -> O(n))
  3. HashMap for dereplicate (classify O(n^2) -> O(n))
  4. HashMap for boths_map (classify O(n) -> O(1) per query)
  5. Fixed array for enumerate_single sliding window

  Phase 3 - Constant-factor:
  6. Byte-level DNA operations (reverse_complement, remove_gaps)
  7. Parallel IDF weight computation (rayon)
  8. sample_int_replace_into for pre-allocated buffers

  Phase 4 - Quick wins:
  9. Release profile: LTO + codegen-units=1
  10. O(n) groupMax single-pass (was O(n*g) brute force)
  11. Flat contiguous hits matrix in parallel_match (was Vec<Vec<f64>>)
  12. Sequential path in parallel_match for small n (avoids rayon overhead)

  Phase 5 - Allocation reduction:
  13. Eliminated sampling.clone() via sort-index approach
  14. Eliminated grouped Vec<Vec<usize>> allocation storm
  15. Eliminated sampling_weights Vec via inline davg computation
  16. Flat hits in tree descent (was Vec<Vec<f64>>)
